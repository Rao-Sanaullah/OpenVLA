What's Fixed Here

Issues not covered in the official OpenVLA docs — encountered on real hardware:

Issue Fix
🔴 RTX 5090 (sm_120) not supported by stable PyTorch Use PyTorch nightly cu128 + set TORCH_CUDA_ARCH_LIST="12.0"
🔴 SSL certificate errors behind corporate firewall Install company cert into image before any pip/git commands
🔴 Old PyTorch cached in Docker layer (still shows cu121) Use docker compose build --no-cache and delete old image first
🔴 Model weights re-downloaded on every container run Mount ./hf_cache and set HF_HOME=/workspace/hf_cache
🔴 Flash Attention 2 warning about GPU init order Load model to CPU first, then .to('cuda:0') — already handled in examples below
01

Prerequisites

Install Docker and the NVIDIA Container Toolkit on your host machine:

bash — host machine 
# Install Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER && newgrp docker

# Install NVIDIA Container Toolkit
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
  sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg

curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
ℹ️
Get your HuggingFace token at huggingface.co/settings/tokens — you need read access. The openvla-7b model is public so no access request needed.
02

Folder Structure

Clone the OpenVLA repo and set up your working directory. Everything lives in one place:

bash 
# Go to wherever you want the project
cd /your/preferred/path

# Clone the OpenVLA repo
git clone https://github.com/openvla/openvla.git
cd openvla

# Create persistent data folders
mkdir -p hf_cache datasets checkpoints outputs

Your final structure:

openvla/ ← your working directory
  ├── prismatic/ model code
  ├── vla-scripts/ training scripts
  ├── experiments/ eval scripts
  ├── hf_cache/ model weights — persistent across runs
  ├── datasets/ robot datasets
  ├── checkpoints/ fine-tuned model saves
  ├── outputs/ logs and results
  ├── Dockerfile ← create this
  ├── docker-compose.yml ← create this
  ├── .env ← create this
  └── company-root.crt ← only if behind corporate firewall
03

Dockerfile

Create Dockerfile inside the openvla/ folder. If you are not behind a corporate firewall, remove the certificate block.

Dockerfile 
FROM nvidia/cuda:12.8.0-cudnn-devel-ubuntu22.04

ENV DEBIAN_FRONTEND=noninteractive
ENV PYTHONUNBUFFERED=1
ENV TORCH_CUDA_ARCH_LIST="12.0"

# ── Corporate firewall certificate (remove if not needed) ────────────────────
COPY company-root.crt /usr/local/share/ca-certificates/company-root.crt
RUN apt-get update && apt-get install -y ca-certificates && \
    update-ca-certificates
ENV REQUESTS_CA_BUNDLE=/etc/ssl/certs/ca-certificates.crt
ENV SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt
ENV GIT_SSL_CAINFO=/etc/ssl/certs/ca-certificates.crt
ENV CURL_CA_BUNDLE=/etc/ssl/certs/ca-certificates.crt
# ─────────────────────────────────────────────────────────────────────────────

# System dependencies
RUN apt-get update && apt-get install -y \
    python3.10 \
    python3.10-dev \
    python3-pip \
    git \
    git-lfs \
    ninja-build \
    build-essential \
    wget \
    curl \
    && rm -rf /var/lib/apt/lists/*

# Make python3.10 the default
RUN update-alternatives --install /usr/bin/python python /usr/bin/python3.10 1 && \
    update-alternatives --install /usr/bin/pip pip /usr/bin/pip3 1

RUN pip install --upgrade pip setuptools wheel

# ── PyTorch nightly cu128 — REQUIRED for RTX 5090 (Blackwell / sm_120) ───────
RUN pip install --pre --upgrade \
    torch \
    torchvision \
    torchaudio \
    --index-url https://download.pytorch.org/whl/nightly/cu128
# ─────────────────────────────────────────────────────────────────────────────

# OpenVLA Python dependencies
RUN pip install \
    "transformers==4.40.1" \
    "tokenizers==0.19.1" \
    "timm==0.9.10" \
    "huggingface_hub" \
    "accelerate" \
    "peft" \
    "draccus" \
    "einops" \
    "pillow" \
    "wandb" \
    "tensorflow==2.15.0" \
    "tensorflow-datasets==4.9.3" \
    "dlimp @ git+https://github.com/moojink/dlimp_openvla"

# Flash Attention 2 — compiled for Blackwell
RUN pip install packaging ninja && \
    pip install "flash-attn==2.5.5" --no-build-isolation

WORKDIR /workspace
⚠️
No corporate firewall? Remove the 6 lines between the certificate comments. Also remove the COPY company-root.crt line and delete company-root.crt from the COPY instruction.
04

docker-compose.yml

Create docker-compose.yml inside the openvla/ folder. The entire repo folder is mounted into the container so edits on your host are instantly reflected.

yaml 
services:
  openvla:
    build: .
    image: openvla:latest
    runtime: nvidia
    environment:
      - NVIDIA_VISIBLE_DEVICES=all
      - NVIDIA_DRIVER_CAPABILITIES=compute,utility
      - HF_TOKEN=${HF_TOKEN}
      - HUGGING_FACE_HUB_TOKEN=${HF_TOKEN}
      - HF_HOME=/workspace/hf_cache
    volumes:
      - .:/workspace                         # live code from host
      - ./hf_cache:/workspace/hf_cache       # model weights — persistent
      - ./datasets:/workspace/datasets       # robot datasets
      - ./checkpoints:/workspace/checkpoints # fine-tuned models
      - ./outputs:/workspace/outputs         # logs and results
    shm_size: '32gb'
    ulimits:
      memlock: -1
      stack: 67108864
    stdin_open: true
    tty: true
    command: bash -c "cd /workspace && pip install -e . --quiet && bash"
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]

.env

⚠️
Add .env to your .gitignore — never commit your HuggingFace token to a public repo.
06

Build the Image

⏱️
First build takes 30–40 minutes. Flash Attention compiles from source (~15 min). PyTorch nightly downloads ~5 GB. Subsequent builds use Docker layer cache and are much faster.
bash 
# Make sure you're in the openvla folder
cd /your/path/openvla

# Build (use --no-cache if rebuilding to fix PyTorch version)
docker compose build

# If you need a completely fresh build (e.g. wrong PyTorch cached)
docker rmi openvla:latest
docker compose build --no-cache
07

Verify GPU & Model

Step 1 — Verify RTX 5090 GPU compute works

bash 
docker compose run --rm openvla python -c "
import torch
print('PyTorch:', torch.__version__)
print('CUDA:', torch.version.cuda)
print('GPU:', torch.cuda.get_device_name(0))
print('Compute capability:', torch.cuda.get_device_capability(0))
x = torch.randn(1000, 1000, device='cuda')
y = torch.mm(x, x)
print('GPU compute test PASSED! Shape:', y.shape)
"

Expected output:

PyTorch: 2.12.0.dev...+cu128
CUDA: 12.8
GPU: NVIDIA GeForce RTX 5090
Compute capability: (12, 0)
GPU compute test PASSED!
No sm_120 warnings

Step 2 — Load the full OpenVLA 7B model (~14 GB download, only once)

bash 
docker compose run --rm openvla python -c "
import torch
from transformers import AutoModelForVision2Seq, AutoProcessor

processor = AutoProcessor.from_pretrained(
    'openvla/openvla-7b', trust_remote_code=True)

vla = AutoModelForVision2Seq.from_pretrained(
    'openvla/openvla-7b',
    attn_implementation='flash_attention_2',
    torch_dtype=torch.bfloat16,
    low_cpu_mem_usage=True,
    trust_remote_code=True
).to('cuda:0')

print('Model loaded!')
print('Parameters:', round(sum(p.numel() for p in vla.parameters()) / 1e9, 2), 'B')
print('GPU memory used:', round(torch.cuda.memory_allocated(0)/1024**3, 2), 'GB')
print('GPU memory free:', round(torch.cuda.mem_get_info(0)[0]/1024**3, 2), 'GB')
"
✅
Expected: 7.54B parameters, ~14 GB VRAM used, ~16 GB free. Weights are cached in hf_cache/ and won't re-download on subsequent runs.
08

Daily Usage

Interactive shell

bash 
docker compose run --rm openvla bash

Run inference script

bash 
docker compose run --rm openvla python vla-scripts/deploy.py \
  --pretrained_checkpoint openvla/openvla-7b \
  --host 0.0.0.0 \
  --port 8000

LoRA fine-tuning

bash 
docker compose run --rm openvla \
  torchrun --standalone --nnodes 1 --nproc-per-node 1 \
  vla-scripts/finetune.py \
  --vla_path openvla/openvla-7b \
  --data_root_dir /workspace/datasets \
  --dataset_name bridge_orig \
  --run_root_dir /workspace/checkpoints \
  --lora_rank 32 \
  --batch_size 16 \
  --learning_rate 5e-4 \
  --image_aug True
💡
Edit code in openvla/ on your host machine with any editor — changes appear instantly inside the container because the folder is mounted as a volume.

Troubleshooting

Error Fix
sm_120 is not compatible + RuntimeError on CUDA op PyTorch is wrong version. Delete image with docker rmi openvla:latest and rebuild with --no-cache
SSL certificate verify failed Add your company cert to the Dockerfile as shown in Step 03. Must be installed before any pip/git command.
Model re-downloads every run Confirm HF_HOME=/workspace/hf_cache is in .env and ./hf_cache is mounted in compose
yaml: mapping values not allowed on docker compose Open docker-compose.yml in a text editor and re-paste cleanly — the file has stray characters
unknown instruction: cat in Dockerfile The cat > shell command got written into the file. Open with nano Dockerfile, delete all, paste the Dockerfile content only
Flash Attention build fails Ensure base image is cuda:12.8.0-cudnn-devel (devel variant). The runtime variant lacks compiler headers.
OOM during fine-tuning Reduce --batch_size and increase --grad_accumulation_steps to compensate. RTX 5090 has 32 GB VRAM.

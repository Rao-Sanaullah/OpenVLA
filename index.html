What's Fixed Here

Issues not covered in the official OpenVLA docs — encountered on real hardware during this install:

IssueFix Applied
RTX 5090 (sm_120 / Blackwell) not supported by stable PyTorch Use PyTorch nightly cu128 and set TORCH_CUDA_ARCH_LIST="12.0"
SSL certificate errors behind corporate firewall Install company cert into the image before any pip or git command runs
Old PyTorch (cu121) served from Docker layer cache Delete old image with docker rmi then rebuild with --no-cache
Model weights (~14 GB) re-downloaded on every container run Mount ./hf_cache and set HF_HOME=/workspace/hf_cache in environment
Flash Attention warning about GPU initialisation order Load model on CPU first, then call .to('cuda:0') — shown in examples below
01

Prerequisites

Install Docker and the NVIDIA Container Toolkit on your host machine before anything else.

bash — host machine
# Install Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER && newgrp docker

# Install NVIDIA Container Toolkit
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
  sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg

curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
ℹ️
Get your HuggingFace token at huggingface.co/settings/tokens with read access. The openvla-7b model is public — no access request needed.
02

Folder Structure

Clone the OpenVLA repo. Your Docker files and data folders all live inside it.

bash
# Clone the OpenVLA repo
git clone https://github.com/openvla/openvla.git
cd openvla

# Create persistent data folders
mkdir -p hf_cache datasets checkpoints outputs

Final structure — Docker files sit alongside the repo code:

openvla/ ← run all docker commands from here
  ├── prismatic/ model code
  ├── vla-scripts/ training & fine-tuning scripts
  ├── experiments/ evaluation scripts
  ├── hf_cache/ model weights — persists across container runs
  ├── datasets/ your robot datasets
  ├── checkpoints/ fine-tuned model saves
  ├── outputs/ training logs and results
  ├── Dockerfile ← create this (Step 03)
  ├── docker-compose.yml ← create this (Step 04)
  ├── .env ← create this (Step 05)
  └── company-root.crt ← only if behind a corporate firewall
03

Dockerfile

Create Dockerfile inside the openvla/ folder using nano Dockerfile. If you are not behind a corporate firewall, remove the certificate block (the 7 lines between the firewall comments).

Dockerfile
FROM nvidia/cuda:12.8.0-cudnn-devel-ubuntu22.04

ENV DEBIAN_FRONTEND=noninteractive
ENV PYTHONUNBUFFERED=1
ENV TORCH_CUDA_ARCH_LIST="12.0"

# ── Corporate firewall certificate (remove block if not needed) ───────────────
COPY company-root.crt /usr/local/share/ca-certificates/company-root.crt
RUN apt-get update && apt-get install -y ca-certificates && \
    update-ca-certificates
ENV REQUESTS_CA_BUNDLE=/etc/ssl/certs/ca-certificates.crt
ENV SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt
ENV GIT_SSL_CAINFO=/etc/ssl/certs/ca-certificates.crt
ENV CURL_CA_BUNDLE=/etc/ssl/certs/ca-certificates.crt
# ─────────────────────────────────────────────────────────────────────────────

# System dependencies
RUN apt-get update && apt-get install -y \
    python3.10 \
    python3.10-dev \
    python3-pip \
    git \
    git-lfs \
    ninja-build \
    build-essential \
    wget \
    curl \
    && rm -rf /var/lib/apt/lists/*

# Make python3.10 the default
RUN update-alternatives --install /usr/bin/python python /usr/bin/python3.10 1 && \
    update-alternatives --install /usr/bin/pip pip /usr/bin/pip3 1

RUN pip install --upgrade pip setuptools wheel

# PyTorch nightly cu128 — required for RTX 5090 (Blackwell / sm_120)
# Stable PyTorch does NOT support sm_120 and will crash at runtime
RUN pip install --pre --upgrade \
    torch \
    torchvision \
    torchaudio \
    --index-url https://download.pytorch.org/whl/nightly/cu128

# OpenVLA Python dependencies (pinned versions from official repo)
RUN pip install \
    "transformers==4.40.1" \
    "tokenizers==0.19.1" \
    "timm==0.9.10" \
    "huggingface_hub" \
    "accelerate" \
    "peft" \
    "draccus" \
    "einops" \
    "pillow" \
    "wandb" \
    "tensorflow==2.15.0" \
    "tensorflow-datasets==4.9.3" \
    "dlimp @ git+https://github.com/moojink/dlimp_openvla"

# Flash Attention 2 — compiled for Blackwell
RUN pip install packaging ninja && \
    pip install "flash-attn==2.5.5" --no-build-isolation

WORKDIR /workspace
04

docker-compose.yml

Create docker-compose.yml inside the openvla/ folder. The entire repo is mounted into the container — edit files on your host and changes appear instantly inside.

yaml
services:
  openvla:
    build: .
    image: openvla:latest
    runtime: nvidia
    environment:
      - NVIDIA_VISIBLE_DEVICES=all
      - NVIDIA_DRIVER_CAPABILITIES=compute,utility
      - HF_TOKEN=${HF_TOKEN}
      - HUGGING_FACE_HUB_TOKEN=${HF_TOKEN}
      - HF_HOME=/workspace/hf_cache
    volumes:
      - .:/workspace                         # entire openvla folder → container
      - ./hf_cache:/workspace/hf_cache       # model weights, persistent
      - ./datasets:/workspace/datasets       # robot datasets
      - ./checkpoints:/workspace/checkpoints # saved fine-tuned models
      - ./outputs:/workspace/outputs         # logs and results
    shm_size: '32gb'
    ulimits:
      memlock: -1
      stack: 67108864
    stdin_open: true
    tty: true
    command: bash -c "cd /workspace && pip install -e . --quiet && bash"
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]

.env File

Create .env inside the openvla/ folder:

.env
HF_TOKEN=hf_your_token_here
HF_HOME=/workspace/hf_cache
⚠️
Add .env to your .gitignore — never commit your HuggingFace token to a public repository.
06

Build the Image

⏱️
First build takes 30–40 minutes. Flash Attention compiles from source (~15 min). PyTorch nightly downloads ~5 GB. Subsequent builds use the Docker layer cache and are much faster.
bash
# Make sure you are inside the openvla folder
cd openvla

# First-time build
docker compose build

# If rebuilding to fix a wrong PyTorch version cached in an old image
docker rmi openvla:latest
docker compose build --no-cache
07

Verify GPU & Model

Step 1 — Verify RTX 5090 GPU compute works

bash
docker compose run --rm openvla python -c "
import torch
print('PyTorch:', torch.__version__)
print('CUDA:', torch.version.cuda)
print('GPU:', torch.cuda.get_device_name(0))
print('Compute capability:', torch.cuda.get_device_capability(0))
x = torch.randn(1000, 1000, device='cuda')
y = torch.mm(x, x)
print('GPU compute test PASSED! Shape:', y.shape)
"

Expected output — all six should be clean with no warnings:

PyTorch: 2.12.0.dev...+cu128
CUDA: 12.8
GPU: NVIDIA GeForce RTX 5090
Compute capability: (12, 0)
GPU compute test PASSED!
No sm_120 incompatibility warning

Step 2 — Load the full OpenVLA 7B model

Downloads ~14 GB on first run, then cached permanently in hf_cache/.

bash
docker compose run --rm openvla python -c "
import torch
from transformers import AutoModelForVision2Seq, AutoProcessor

processor = AutoProcessor.from_pretrained(
    'openvla/openvla-7b', trust_remote_code=True)

vla = AutoModelForVision2Seq.from_pretrained(
    'openvla/openvla-7b',
    attn_implementation='flash_attention_2',
    torch_dtype=torch.bfloat16,
    low_cpu_mem_usage=True,
    trust_remote_code=True
).to('cuda:0')

print('Model loaded!')
print('Parameters:', round(sum(p.numel() for p in vla.parameters()) / 1e9, 2), 'B')
print('GPU memory used:', round(torch.cuda.memory_allocated(0)/1024**3, 2), 'GB')
print('GPU memory free:', round(torch.cuda.mem_get_info(0)[0]/1024**3, 2), 'GB')
"
✅
Expected: 7.54B parameters, ~14 GB VRAM used, ~16 GB free on the RTX 5090 (32 GB total). Weights are cached and won't re-download on future runs.
08

Daily Usage

Interactive shell

bash
docker compose run --rm openvla bash

Serve OpenVLA over REST API

bash
docker compose run --rm openvla python vla-scripts/deploy.py \
  --pretrained_checkpoint openvla/openvla-7b \
  --host 0.0.0.0 \
  --port 8000

LoRA fine-tuning (single GPU)

bash
docker compose run --rm openvla \
  torchrun --standalone --nnodes 1 --nproc-per-node 1 \
  vla-scripts/finetune.py \
  --vla_path openvla/openvla-7b \
  --data_root_dir /workspace/datasets \
  --dataset_name bridge_orig \
  --run_root_dir /workspace/checkpoints \
  --lora_rank 32 \
  --batch_size 16 \
  --learning_rate 5e-4 \
  --image_aug True
💡
Edit any file in the openvla/ folder on your host machine with any editor — changes appear instantly inside the container because the folder is mounted as a volume.

Troubleshooting

ErrorFix
sm_120 is not compatible + RuntimeError on any CUDA op PyTorch is the wrong version. Run docker rmi openvla:latest then docker compose build --no-cache
SSL certificate verify failed during build Add your company cert to the Dockerfile as shown in Step 03. It must be installed before any pip or git command.
Model weights re-downloaded every container run Confirm HF_HOME=/workspace/hf_cache is in .env and ./hf_cache is in the volumes list in compose
yaml: mapping values not allowed on docker compose Open docker-compose.yml in a text editor (nano docker-compose.yml), delete everything, and paste the YAML from Step 04 cleanly
unknown instruction: cat in Dockerfile A shell command got written into the file. Open with nano Dockerfile, delete all content, paste only the Dockerfile lines from Step 03
Flash Attention build fails Ensure the base image is cuda:12.8.0-cudnn-devel-ubuntu22.04 — the devel variant is required. The runtime variant lacks compiler headers.
Out of memory during fine-tuning Reduce --batch_size and increase --grad_accumulation_steps proportionally. The RTX 5090 has 32 GB VRAM.
